# Prospective field-pilot guide

This guide is a prerequisite for operational claims. It does not authorize the
system to identify adulterants or make adverse decisions from screening output.

## Objective

Measure whether screening signals improve the speed and yield of controlled
resampling and confirmatory testing without creating unacceptable false alarms.

## Minimum protocol

1. Select representative facilities, instruments, seasons, shifts, and milk
   classes before observing results.
2. Freeze the engine version, workbook contract, thresholds, and review process.
3. Record input-validation results, accepted/rejected rows, and every case
   disposition—not only signals that later appear correct.
4. For selected signals, record chain-of-custody identifiers, reference method,
   result, turnaround time, and an outcome category such as `confirmed`,
   `instrument_or_process_issue`, `normal_variation`, or `inconclusive`.
5. Use qualified laboratory methods appropriate to the suspected condition; do
   not infer a substance solely from fat, SNF, CLR, or quantity.

## Report before deployment

- Signal rate and false-positive rate per record and producer-period.
- Sensitivity, specificity, predictive values, and uncertainty intervals where
  reference outcomes are available.
- Performance stratified by facility, instrument, operator, season, milk class,
  and relevant operational groups.
- Parser rejection rate, data drift, shared-event rate, and time to disposition.
- Any change-control decision, including the exact version that produced each
  result.

Only an independently reviewed pilot with adequate reference coverage can
support a claim about operational effectiveness.

# Security Policy

## Reporting a vulnerability

Do not disclose suspected vulnerabilities in a public issue. Use GitHub's
private vulnerability reporting feature for this repository.

Include the affected component, reproduction steps, potential impact, and any
suggested mitigation. Do not include real customer data or active credentials.

## Data handling

This repository must contain only synthetic or explicitly redistributable
fixtures. Deployments are responsible for access control, encryption, data
retention, tenant isolation, audit logging, and applicable legal obligations.

No screening result should be treated as laboratory confirmation or used as
the sole basis for an adverse decision.

# Contributing

Thank you for helping improve Milk Quality Screening.

## Ground rules

- Never commit real customer, farmer, supplier, vehicle, or facility data.
- Use synthetic names and generated measurements in tests and examples.
- Keep claims scientifically bounded to screening and prioritization.
- Add tests for behavior changes and document methodology changes.
- Do not include secrets, credentials, private URLs, or production exports.

## Development workflow

The installable package and contributor commands will be documented when the
engine extraction lands. Until then, open an issue before beginning a large
change so the work aligns with the public architecture.

## Pull requests

Describe the problem, the chosen approach, validation performed, and any
methodology or privacy implications. Keep unrelated changes separate.
